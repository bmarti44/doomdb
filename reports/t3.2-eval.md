## Candidate delivered

- Fourteen stable visible test IDs with 136 declared assertions and fourteen
  named semantic mutations.
- An independently decoded pinned-WAD oracle for all vertex extrema, the
  absence of degenerate lines, and six directed length/vector probes spanning
  oblique, vertical, horizontal, shortest, and longest linedefs.
- A closed schema contract for directed two-point `SDO_GEOMETRY`, null SRID,
  canonical element info, validity tolerance `0.005`, and decimal metrics
  rounded to twelve places using the unrounded Euclidean norm.
- Live configuration values `FAR_DISTANCE=8192` and `PLAYER_RADIUS=16`, with
  metadata required to derive its bounds from database `MIN/MAX` plus their
  sum. The known resulting bounds are checked as output but forbidden as
  production SQL literals.
- R-tree checks covering domain-index type and Oracle base, domain, and operator
  status rather than accepting index presence alone.
- An isolated Oracle mini-map with a real diagonal-line MBR false positive:
  `SDO_FILTER` returns one candidate and the following exact `SDO_RELATE`
  predicate returns zero. Positive contact and large-translation cases guard
  against an evaluator that merely rejects everything.
- A pinned E1M1 false-positive probe against deployed linedef zero, plus a
  fail-closed production source audit requiring every `SDO_FILTER` statement to
  contain a subsequent exact geometry predicate.
- A disposable-schema runner that refuses to place mini-map objects in the
  production schema and removes the evaluator schema on every exit path.

The exact-predicate mutation counts as killed only when the database and R-tree
remain healthy and the named false-positive assertion observes the unwanted
row. Compile failures, invalid indexes, empty discovery, and incidental errors
do not count.
