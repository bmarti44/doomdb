## Frozen deployment contract

The gate accepts only the explicit HTTPS `index.html` object on one attested S3
REST origin and direct dynamic calls to one attested Oracle managed ORDS origin.
Local hosting, S3 website HTTP, CloudFront, proxies, service workers, mocked
routes, prerecorded traces, dry runs, and source-only evidence fail closed.

The live upload inventory must equal a deterministic compiled-client allowlist.
Every object is verified by live HEAD and GET against its source SHA-256 and has
an exact content type. `index.html` always revalidates; year-long immutable
caching is accepted only when the bundle filename digest prefixes the uploaded
byte hash. Source maps, runtime configuration, source, tools, evaluators,
fixtures, reports, goldens, WAD inputs and secrets are forbidden.

The managed ORDS base URL is embedded at build time. A pinned one-worker
Playwright 1.61.0 Chromium run from the actual S3 URL must observe a real CORS
OPTIONS exchange and cover new-game, STEP, PLAYPAL/audio assets, exact 320x200
canvas bytes, database audio events, save/load, replay and completion smoke. Its
complete network ledger may contain only the attested S3 and Oracle origins.

AWS and ADB credentials, bucket/region/account names, endpoints, authorization,
game tokens and session identifiers are forbidden from retained evidence. Only
SHA-256 identities and redacted observations survive, published atomically after
every live operation succeeds.
