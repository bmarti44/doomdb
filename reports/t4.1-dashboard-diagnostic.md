# T4.1 live dashboard diagnostic

`sql/render/r1/diagnostics/010_dashboard_rays.sql` is a read-only, bounded
review query for the live dashboard. Bind `:p_session` to an existing session
token. It returns one compact JSON document containing exactly 320 real camera
columns, per-column analytic hit counts, and each nearest static solid hit.

The payload is database-authored from `DOOM_R1_RAYS`, `DOOM_R1_HITS`, and
`DOOM_R1_NEAREST`; it does not create a session, mutate a pose, use a fixture,
or supply fallback/mock values. It is therefore safe to expose through the
eventual reviewed read-only diagnostic surface without moving rendering logic
into the dashboard.

Example document shape (field names only, not mock values):

```json
{
  "schema": 1,
  "kind": "doomdb-r1-ray-diagnostic",
  "width": 320,
  "columns": [
    {
      "column": "number",
      "camX": "number",
      "rayX": "number",
      "rayY": "number",
      "hitCount": "number",
      "nearestT": "number",
      "nearestU": "number",
      "linedefId": "number",
      "segId": "number",
      "facingSide": "number"
    }
  ]
}
```
